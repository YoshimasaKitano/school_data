# 図書館管理システム（CLI）について
## 実装課題：学習レベル判定基準

この図書館管理システム（CLI）の実装は、「中級レベル」に相当します。  

### 必要なスキルと学習内容

| スキル領域 | 必要な内容 | 難易度 |
|------------|------------|--------|
| **Python基礎** | 変数、関数、条件分岐、ループ、リスト・辞書操作 | 初級 |
| **ファイル操作** | JSONファイルの読み書き（`json`モジュール） | 初級〜中級 |
| **関数設計** | 各機能を関数に分割し、引数と戻り値を適切に設計 | 中級 |
| **例外処理** | `try-except`によるエラーハンドリング | 中級 |
| **CLI設計** | `input()`と`print()`を使ったメニュー制御 | 初級〜中級 |
| **データ構造の理解** | ネストされた辞書やリストの操作 | 中級 |
| **状態管理** | ユーザーごとの貸出状態を保持・更新 | 中級 |
| **テスト性の考慮** | 関数単位でのテスト設計 | 中級〜上級（任意） |


### 中級とするポイント

- **複数ファイルのデータ管理（JSON）**：ファイルの読み書きとデータの永続化が必要。
- **状態遷移の管理**：貸出・返却処理では状態の整合性を保つ必要がある。
- **例外処理の設計**：ユーザー入力のバリデーションやエラー時のメッセージ表示が求められる。
- **CLIメニューの制御**：階層的なメニュー構造を制御するロジックが必要。


### 上級ではない理由

- 設計書等の作成はなく、実装のみである。
- GUIやWebフレームワーク（Flask/Django）などの使用はない。
- データベース（SQL）との連携されていない。
- クラスベースの設計（OOP）ではなく、関数ベースで完結している。
- 外部ライブラリや非同期処理などの高度な技術は含まれていない。
