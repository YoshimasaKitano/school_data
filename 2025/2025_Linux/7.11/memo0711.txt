ディレクトリのパーミッション
　x 実行権限がないとカレントディレクトリとして選択できない
　  ディレクトリ配下にファイルやディレクトリを作成できない。

 r 4 読み取り
 w 2 書き込み
 x 1 実行

　例：オーナーに読み書き実行、グループに読み取り、その他のユーザーに読み取り

　chmod 744 ファイル名
　chmod u=rwx, go=r ファイル名
　chmod u=rwx, g=r, o=r ファイル名

　★スーパーユーザ(管理者ユーザ=root)
　　管理者権限を持つ特別なユーザー

   sudo コマンド(推奨)：一般ユーザが管理者権限でコマンドを実行できる

　　例：sudo shutdown -r now

　　→sudoを使えるようにする
　　/etc/sudoers ファイルにユーザ名と権限を追加する
　　または、wheelグループにユーザを追加する

　　sudo usermod -aG wheel username #グループへの追加方法

   su コマンド：ユーザを切り替える
　　　※exitするまで切り替えたユーザのまま

　　　シェルからみたコマンドの実行単位

　
　★ジョブ管理
　　　jobs シェルで実行中のコマンド一覧を表示

　　　Ctrl+Z ジョブの一時停止

　　　fg %ジョブ番号 指定したジョブをフォアグラウンド(前面)で実行
　　　bg %ジョブ番号 指定したジョブをバックグラウンド(背面)で実行

　★プロセス、ジョブの終了
　　　kill -9 [PID] 指定したプロセスまたはジョブを強制終了
　　　kil -9 %ジョブ番号

　　　kill -15 [PID] 指定したプロセスまたはジョブを終了
　　　kill -15 %ジョブ番号

　　　kill [PID] 指定したプロセスまたはジョブを終了(-15を指定したのと同じ)

標準入力→標準出力
	→標準エラー出力

リダイレクト：出力先を変更すること
　コマンドの実行結果をファイルに出力＝ファイルにリダイレクト

  上書きモード/追記モード

 ls -l / > list.txt　#上書きモード
 ls -l / >> list.txt　#追記モード

 標準エラー出力のリダイレクト

 ls -l / 2> list.txt　#上書きモード
 ls -l / 2>> list.txt　#追記モード

入力リダイレクト
　例： cat < ファイル名

★特殊なファイル
 /dev/null
    ・入力元として指定しても、何も内容を返さない
　　・出力先として指定しても、書き込んだデータはどこにも保存されず消えてなくなる

　利用シーン：標準出力と標準エラー出力が同時に出力される環境下で、エラーメッセージだけを表示する。またはその逆。

----------------------------------------
  パイプライン…コマンドの標準出力を次のコマンドの標準出力にする

　書式； <コマンド1> | <コマンド2>

　lsの実行結果に、行番号を追加して、逆順に表示する

　ls -l | cat -n | tac

フィルタ
　head		最初の10行を表示
　head -n 数字	最初から数字分の行数を表示
　head -数字	最初から数字分の行数を表示
　tail		最後の10行を表示
　tail -n 数字	最後から数字分の行数を表示
　tail -数字	最後から数字分の行数を表示

　cat		入力をそのまま出力する
　cat -n	行番号をつけて出力
　tac		逆順に出力

　uniq		重複行を排除して出力する
　sort		順番に並び変えて出力する
　sort -n	数字の昇順に並べて出力する
　sort -r	数字の降順に並べて出力する
　sort -k	フィールド番号		フィールド番号の項目で並びかえる

	例： ls -l | sort -k 5 #ファイルの容量(第5フィールド)でソート

　grep		パターン検索し一致した行を出力する



　wc		ファイルの行数、単語数、バイト数を表示
　wc -l		行数を数える
　wc -w		単語数を数える
　wc -c		半角文字数(Byte)を数える
